
Após a instalação e funcionamento correto do Immich, tivemos que agrupar e upar nossas fotos, em nossas nuvens como Google fotos e outras plataformas haviam grandes quantidades de fotos e videos dos quais foram baixados em .zip com seus devidos metadados.

### O Método "Oficial": Immich-CLI (Recomendado para grandes volumes)

A equipe do Immich criou uma ferramenta de linha de comando (CLI) feita especificamente para importar o **Google Takeout**.

- **Vantagem:** Ela preserva todas as datas originais, localizações (GPS) e metadados que o Google costuma "separar" em arquivos `.json`.
    
- **Como funciona:**
    
    1. Você solicita o backup no [Google Takeout](https://takeout.google.com/) (selecione apenas o Google Fotos).
        
    2. Baixa os arquivos `.zip` para o seu computador.
        
    3. Usa o comando de upload do Immich apontando para esses arquivos.
       
   ### Usando o Immich-Go (Mais Rápido e Eficiente)

O **Immich-Go** é uma ferramenta de linha de comando que pode "varrer" o seu disco interno e subir tudo em lote. No terminal do Athena (fora do chroot), faça o seguinte:

1. **Descarregue a ferramenta:**
    
    Bash
    
    ```
    wget https://github.com/simulot/immich-go/releases/latest/download/immich-go_Linux_x86_64.tar.gz
    tar -xf immich-go_Linux_x86_64.tar.gz
    chmod +x immich-go
    ```
    
2. **Execute o Upload:** Substitua `API_KEY` pela chave que você gera no Immich (em _Settings > API Keys_) e aponte para a pasta das fotos no disco montado:
    
    Bash
	
    ```
./immich-go upload from-google-photos LOCAL-ARQUIVO -s SERVIDOR -k SUA_API_KEY
