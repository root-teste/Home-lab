Como não queremos que nosso servidor se "perca na rede" vamos configurar um ip fixo.

Optou-se pelo uso de ARP estático no servidor para garantir o mapeamento fixo entre endereço IP e MAC, aumentando o controle e a segurança da rede. Diferente do DHCP, que automatiza a distribuição de endereços IP, o ARP não realiza atribuição de IP, mas assegura a integridade das associações na rede local.  
Em ambientes domésticos ou roteadores comerciais, é comum encontrar apenas DHCP (com ou sem reservas), pois oferece maior simplicidade e escalabilidade, enquanto o uso de ARP estático exige configuração manual e é mais adequado a cenários controlados.

A configuração pelo roteador é simples, sendo feita apenas com o endereço IP e mac desejado, a configuração é guardada no cache o que já ajuda no controle e mitigação de vulnerabilidades futuras.

O acesso é feito de forma simples pelo gatware padrão do roteador, muitos possuem outras configurações que podem e devem ser exploradas para maior proteção de nosso servidor.
